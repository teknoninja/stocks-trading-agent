A free-tier AI trading assistant. Deterministic maths make the BUY / SELL / HOLD decision; an LLM only explains it. The same engine serves a live browser sidebar and an autonomous cloud scanner. Follow the colour of each node to see the job it does.

3 · The life of one trade

A single RELIANCE position traced through every layer — the exact value at each hop.

  1. You open NSE:RELIANCE

    detectSymbol() returns the symbol; the sidebar calls /analyze via the background worker.

  2. Resolve & fetch

    data.py maps it to RELIANCE.NS, fetches four timeframes. Current price ₹1,327.

  3. Strategies vote

    Weekly + daily structure up, golden cross, bullish BOS — outweigh one weekly RSI divergence.

  4. Aggregate → flag

    scoring.py: score +0.28 ≥ 0.22 → BUY, confidence 62%.

  5. Scanner buys

    Confidence ≥ 0.55, not held, room free → virtual_broker.buy() takes 1 share @ ₹1,327, stamps opened_at.

  6. Holds across days

    Day 2: +1.0%, day 1/7 → hold. Day 4: −0.5% → hold. Log: holding (pnl +1.0%, day 1/7).

  7. Take-profit fires

    Day 6 hits ₹1,395 → pnl +5.1% ≥ 5% → exit_reason returns take_profit. Sold, booking ₹68.

  8. Dashboard updates

    /performance counts ₹68 in realized_pnl; the equity curve rises. Zero manual steps after step 5.

4 · Where state lives

The scanner keeps no memory between runs — it re-derives everything from two sources of truth it can always reach. That is why the identical code runs on your laptop or a throwaway cloud machine.
